CSS columns approach: each column fills top-to-bottom. No inter-row gaps — cards pack tightly regardless of height differences. Works in all modern browsers with zero JS.
Trade-off: reading order goes top→bottom per column, not left→right across the row. This can feel unintuitive when content has a natural sequence. A JS-based masonry (absolute positioning) would preserve row order but adds complexity.
When this shines: description snippets, fun facts, variable badge counts, or mixed photo/no-photo cards — anywhere card height naturally varies. Less useful if all cards stay the same height.

No photos: each row is ~52 px tall — 8 species visible without scrolling vs. 4 in grid view. Ideal for power users, large datasets, or comparing many species quickly.
Two distinct actions: the species name (underlined, green) opens the full modal — same as clicking a card in grid view. The ▸/▾ chevron on the right independently toggles the inline expand. Row background hover is intentionally subtle to avoid implying the whole row is clickable.
Column headers double as sort triggers (name A→Z, health ↑↓, threat severity). Sortable list is much simpler to implement here than in the grid/masonry layouts.
Summary — when to use each layout
Grid (current)
✓ Ordered rows, natural left→right scan
✓ Photos prominent — good first impression
✓ Simple CSS, no JS needed
✗ Blank space below short cards
✗ Only ~4 species visible at once
✗ Hard to compare stats across cards
Masonry
✓ No wasted space — cards pack tightly
✓ Feels rich and dynamic
✓ Photos still present, visual appeal
✗ Reads top→bottom per column
✗ Hard to compare cards in same "row"
✗ Filter transitions can look jarring
List
✓ Most compact — 8+ species visible at once
✓ Easy to compare stats across species
✓ Sortable columns (name, health, threat)
✓ Expandable rows for deeper detail
✗ No photos — less visually engaging
✗ More complex to implement (column layout)
Recommendation: all three work well as user-toggled options. Grid is the right default. Masonry adds value if cards gain variable content (descriptions, fun facts). List is the power-user mode — especially useful once there are 20+ species and users want to sort or scan stats quickly.
